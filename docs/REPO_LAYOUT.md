# Repository layout

Paths are relative to the project root. **Suggested mental order:** core library → weights → corpora → analysis → documentation → CLIs → supplementary GPU code → paper outputs → tests → **archive last**.

## Top-level

| Path | Role |
|------|------|
| `pipeline/` | Library: `graph.py` (Neo4j + grounding), `planning.py`, `systems/` (S1–S5), `metrics/` (CCR, HTM, corpus scores). Import as `from pipeline…`. |
| `models/` | Checkpoints / merged exports — not in git ([models/README.md](../models/README.md)). |
| `data/` | Segment JSONLs, ontology exports, optional `test_data/` PDFs ([data/README.md](../data/README.md)). |
| `error_analysis/` | Generated annotation CSVs ([error_analysis/README.md](../error_analysis/README.md)); not primary data. |
| `docs/` | Layout (this file), error schema, notes ([docs/README.md](README.md)). |
| `tools/` | CLI entrypoints grouped by role — **[`tools/README.md`](../tools/README.md)** (`data/`, `pipeline/`, `eval/`, `error_analysis/`, `admin/`). |
| `extras/` | Supplementary NER, Unsloth training, vector grounding ([extras/README.md](../extras/README.md)); code under `extras/experiments/`. |
| `results/` | **`ner_biollm/`**, **`ner_biollm_finetuned/`**, `cross_ner_comparison/`, `htm_vector_comparison/` (regenerated by `rerun_all.sh` unless skipped). Translation JSONLs may be gitignored. |
| `tests/` | `pytest`. |
| `archive/` | **Last / retired:** old reruns, snapshots, large logs — local only (see root `.gitignore`). Prefer run logs under `archive/logs/` when you keep a tree. |
| `experiments/` | Stub pointer only ([experiments/README.md](../experiments/README.md)) — use `extras/` for real code. |
| `rerun_all.sh` | Batch driver: **BioMistral prompt + FT BioMistral NER**; eval, cross-NER dashboard, HTM threshold step. **Excludes segment `48_028`** unless `EXCLUDE_SEGMENT_IDS=` is empty. |

## `tools/` layout

Scripts live under **`tools/data/`** (corpora + graph + ontology exports), **`tools/pipeline/`** (S1–S5 JSONL runs), **`tools/eval/`** (metrics + figures), **`tools/error_analysis/`** (audits + annotation helpers), and **`tools/admin/`** (snapshot tarball). See **[`tools/README.md`](../tools/README.md)** for the full table.

## Design

Importable code lives in **`pipeline/`**. **`tools/`** holds runnable CLIs only (grouped in subfolders above). Supplementary training and audits live under **`extras/experiments/`** (see **`extras/README.md`**).
