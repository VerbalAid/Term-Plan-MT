3 Metrics

HTM — Hierarchy-Aware Terminology Metric
HTM = mean(sᵢ) over NER-grounded spans
sᵢ = 1.0 · exact MedDRA level match
sᵢ = 0.5 · same branch, wrong level
sᵢ = 0.0 · different branch / absent
1.0 exact level 0.5 same branch 0.0 absent
rHTM — reference ceiling

HTM applied to the professional human reference as hypothesis. Measures how well certified translators use MedDRA labels.

HRA — Hyp–Ref Agreement

Fraction of segments where system terminology choice matches the professional reference, independent of MedDRA correctness.

CCR — Concept Coverage Rate

Grounded NER spans ÷ total extracted spans. Dataset-level; identical across all systems within a NER condition.

Document BLEU / COMET (optional)

BLdoc = macro mean of corpus BLEU per document (segment hyps/refs concatenated). COMET = reference-based DA when evaluate.py / plot_figures.py --comet runs with Unbabel COMET installed.

5 Key Findings

①
CCR is the Binding Constraint
+≈7 BLEU (graph); S6 oracle
NER quality unlocks graph fluency (paired bootstrap: strictly positive Δ for S3–S5-Mistral). S6 (gold glossary, same logit boost as S5) isolates whether MedDRA labels vs the mechanism cap HTM — run after hand-reviewed gold_glossary.json.
②
HTM–Practice Gap
rHTM = 0.13–0.16 on human ref.
The professional reference scores lower than several MT systems on HTM. Alias expansion T1→T2→T3 produces zero movement. MedDRA labels are designed for pharmacovigilance coding, not regulatory prose — the gap is structural, not a coverage issue.
③
Consistency Paradox
drift = 0.333 for S3/S4/S5
Graph systems produce lower cross-document consistency than graph-free systems. Mechanism: ambiguous French terms match multiple MedDRA concepts → planning stage abstains from locking → per-segment context-dependent resolution → drift. S1-NLLB's consistency is a vocabulary artefact, not reasoning.
④
HRA–HTM Inversion
S3 HRA=0.59 vs S2 HRA=0.66
Graph systems score higher on HTM yet lower on HRA. Optimising toward MedDRA formal labels diverges from how certified translators actually write. For regulatory MT in practice, HRA is the more meaningful target. HTM measures MedDRA-proximity, not translation quality.
Deployable pattern. S6’s stack — curated FR→EN glossary + PhraseLogitBoost (+ optional force_words_ids) — is the same mechanism as S5 with a domain glossary instead of MedDRA. Suitable for regulated domains (finance, legal, pharma) where terminology is fixed and auditable.
Conclusion  ·  CCR limits graph utility at low coverage. S6 answers a finer question: if oracle glossary phrases beat S5 on HTM, MedDRA term choice is the residual bottleneck; if S6 ≈ S5, the logit-boost channel is too weak. Run run_pipeline.py --system s6 --glossary data/section48/gold_glossary.json --s6-backend both after building the glossary.
